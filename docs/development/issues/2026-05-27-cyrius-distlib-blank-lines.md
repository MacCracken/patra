# cyrius distlib — generated bundle has "multiple consecutive blank lines" (cyrlint)

**Filed:** 2026-05-27 (during patra 1.10.x lint review, post-1.10.3)
**Cyrius version observed:** 6.0.3 (`cyrius distlib` + `cyrius lint`)
**Tool at fault:** `cyrius distlib` (bundle generator) — surfaced by `cyrius lint`
**Severity:** LOW — cosmetic. The generated bundle is valid and compiles; the
only effect is `cyrlint` warnings on the artifact.
**Repro:** `cyrius distlib && cyrius lint dist/patra.cyr`

## Summary

`cyrius distlib` emits a bundle (`dist/patra.cyr`) that trips cyrlint's
**"multiple consecutive blank lines"** rule. On patra 1.10.3 (bundle = 5130
lines) there are 3 such warnings:

```
$ cyrius lint dist/patra.cyr
  warn line 6: multiple consecutive blank lines
  warn line 3301: multiple consecutive blank lines
  warn line 3304: multiple consecutive blank lines
3 warnings
```

This is notable because **every individual source module lints clean** —
`cyrius lint src/*.cyr` is 0 warnings across all 11 modules. The warnings are
introduced *by the bundling transform itself*, in two distinct ways.

## Root cause

### 1. Header → first-module separator (bundle line 6)

distlib prepends a 4-line header, then emits a blank, then the first module's
`# --- <file> ---` marker preceded by its own blank — yielding two consecutive
blank lines:

```
1  # patra.cyr -- bundled distribution
2  # Version: 1.10.3
3  # Generated by: cyrius distlib
4  # Do not edit -- rebuild with: cyrius distlib
5  (blank)
6  (blank)        <-- warn: consecutive blank
7  # --- file.cyr ---
```

This one is purely distlib's output format — no source file contributes it.

### 2. `include`-line stripping leaves orphaned blanks (bundle lines 3301, 3304)

distlib inlines the modules and strips the `include` lines from the entry file
(`src/lib.cyr`). But it removes only the `include` lines, not the blank lines
that surrounded them, so a `# comment` + blank + `<includes>` + blank +
`# comment` layout collapses to `# comment` + blank + blank + `# comment`.

`src/lib.cyr` (source, lints clean):

```
# ── Stdlib ────────────────────
(blank)
include "lib/syscalls.cyr"
... 9 includes ...
include "lib/sakshi.cyr"
(blank)
# ── Modules (order matters: sql before where before table) ──
(blank)
include "src/file.cyr"
... 10 includes ...
include "src/jsonl.cyr"
(blank)
# ── DB handle (16 bytes) ────────────────────
```

After distlib strips the `include` lines, the two surviving blanks become
adjacent (bundle lines 3300–3301 and 3303–3304):

```
# ── Stdlib ────────────────────
(blank)
(blank)        <-- warn
# ── Modules (order matters ...) ──
(blank)
(blank)        <-- warn
# ── DB handle (16 bytes) ────────────────────
```

## Proposed fix (cyrius distlib)

Collapse runs of ≥2 blank lines to a single blank when writing the bundle —
applied to the final concatenated output, this covers both causes at once
(the header separator and the include-strip residue). Equivalently: when
stripping an `include` line, also consume an immediately-adjacent blank so no
double-blank is left behind, and emit a single blank after the header.

This mirrors prior distlib polish (e.g. the v5.7.36 per-module read-buffer
bump) — the generator should produce lint-clean output since the bundle is the
artifact consumers vendor.

## Impact on patra

- **CI unaffected.** patra's hard lint gate (added in 1.10.1) scans
  `src/*.cyr` + `programs/*.cyr` only — not `dist/`. So this does not fail
  patra's build.
- **Downstream consumers.** A consumer that vendors `dist/patra.cyr` and runs
  a strict lint over its tree (the same posture patra itself takes) will see
  these 3 warnings on the bundle. Cosmetic, but it muddies a clean-lint policy.

## Workaround (available, intentionally NOT applied)

Tidying `src/lib.cyr` to drop the blank between each `# ──` section comment and
its first `include` would eliminate causes (2) — i.e. 2 of the 3 warnings —
because the stripped result would then leave a single blank between section
comments. The header double-blank (cause 1) is pure distlib and unfixable from
patra's source. We are **not** applying the workaround: it's a cosmetic edit to
durable source to paper over an upstream generator bug; the clean fix is the
blank-collapse in distlib above. Re-run `cyrius lint dist/patra.cyr` after the
distlib fix lands and archive this note when it reports 0 warnings.
